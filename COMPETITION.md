# 竞赛约束与作品合规说明

本文档将常见竞赛条款（**国内模型、禁微调、依赖声明、可复现、反作弊**）与 SuperGenius 仓库逐条对应，便于在交付物中引用。

## 1. 模型限制（国内模型、禁止微调）

| 要求 | 在本项目中的落实方式 |
|------|----------------------|
| 统一使用**国内**大语言模型，不得使用境外模型服务 | 运行与评测时，在 `.env` 中将 `LLM_BASE_URL` / `LLM_API_KEY` / `LLM_MODEL` 配置为**国内厂商**提供的端点，例如：火山引擎方舟/豆包、阿里云通义、百度文心、智谱等**官方公开 API**（须为推理 API，与竞赛允许清单一致，以赛方最终规则为准）。 |
| 禁止任何形式的微调 | 本仓库**不包含**任何训练脚本、LoRA/PEFT/RLHF 相关代码与数据；能力增强仅通过 **提示词、RAG、工具调用与 Agent 编排** 实现。 |
| 允许 RAG / 提示词 / 工具与编排 | 与当前架构一致：Agent 内为 Prompt + 可选后续扩展 RAG/工具。 |

**交付物中建议写一句**：「系统推理阶段仅调用（填写具体厂商与模型/端点名称）的**预置推理服务**，未对任何基座模型做微调。」

## 3. 复现与反作弊（真实 CLI/SDK/API；禁止「假装跑通」）

赛规中「**严禁模拟（Mock）任何功能、伪造日志或虚构结果**」在工程上应理解为：

- **被评测/被演示的主流程**须通过**真实** HTTP 与真实凭证完成，例如：真实调用飞书多维表格 API、真实调用国内 LLM 推理端点、真实写入表格并在界面或接口中可核验。
- 仓库中 **pytest** 里使用的内存表、Stub LLM 等，仅属于**开发阶段单元测试**（验证逻辑分支，不对外宣称「已替代生产」）。**竞赛验收与录像演示不应仅依赖 `pytest` 作为「系统已运行」的唯一证据**；应以「执行 [`scripts/`](scripts/) 下脚本 + 在飞书/方舟控制台可见请求与表格变更」为复现主证据。

**推荐可复现步骤（与 README「快速开始」一致）：**

1. 配置环境变量与权限（见 [`.env.example`](.env.example)）
2. `python scripts/bootstrap_tables.py`（真实建表并写入 `.supergenius/tables.json`）
3. `python scripts/seed_jobs.py`
4. `python scripts/run_mvp.py`（保持运行，观察真实多轮调用）
5. `python scripts/candidate_emitter.py`（或等价投递方式）

**建议附加**：环境说明（OS、Python 版本）、网络能访问的域名（飞书、方舟/通义等），以便评测方按相同步骤复现。

## 4. 与「本仓库内测试用 Mock」的关系（说明给评委/选手）

- **生产/演示路径**：[`src/supergenius/`](src/supergenius/) 在运行时通过 `lark_oapi` 与 `LLM_BASE_URL` 指向的推理服务，**不**使用 `tests/conftest.py` 中的 `MemoryBitable` / `StubLLM`。
- **测试路径**：`tests/` 为自动化回归，**不**替代对真实 API 的联调与录制。

若赛方要求「全链路不得使用 Mock」，一般含义是：**提交与评测的演示脚本、日志与表数据必须来自真实调用**；选手应在说明材料中显式写清上述分工，避免被误解为「用测试替身冒充线上」。
